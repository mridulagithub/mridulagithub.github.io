Selected work

Natural-language infrastructure, in Slack
AI Ops · Kubernetes · GitHub · AWS

Problem. Engineers lost time context-switching into consoles to answer routine questions about infrastructure state.

What I built. An AI-powered Slack bot that answers plain-English questions about Kubernetes, GitHub, and AWS and returns results directly in the channel — read-only by design, so it informs without taking actions.

Result. Routine investigation became a quick question instead of a console dive, meeting engineers where they already work.

Reliability as an operating model
SLOs · Observability · Incident response

Problem. Incident response was slow and inconsistent, with no shared reliability language across teams.

What I did. Introduced SLIs, SLOs, KPIs, and error budgets; standardized observability on Prometheus / Grafana; migrated incident management to incident.io with disciplined post-incident analysis.

Result. 80% reduction in MTTR and a reliability practice the whole organization shares.

A platform ready for audit
Governance · Security · Zero-trust

Problem. Achieving SOC 2 Type II required provable governance across the platform and its access paths.

What I did. Led platform readiness, technical auditing, and governance; engineered Just-In-Time production database and VPN access to enforce zero-trust while keeping support fast.

Result. SOC 2 Type II certification achieved, with access controls that accelerate rather than block engineers.
